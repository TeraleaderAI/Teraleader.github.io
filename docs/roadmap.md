# Roadmap

## 우선순위 1: 콘텐츠 정확도 보정

### 회사 정보 확인

현재 회사 정보는 구버전 홈페이지 기준이다.

확인 필요:

- 최신 주소
- 전화번호
- 팩스번호
- 대표 이메일
- 회사명 표기
- 사업자 정보 노출 필요 여부

현재 파일:

```text
site/src/data/company.json
```

### 제품 정보 검수

구버전 상세 페이지에서 추출한 정보를 기준으로 제품 설명과 사양을 구성했다. 실제 최신 제품 정보와 다를 수 있으므로 내부 검수가 필요하다.

확인 필요:

- 제품명
- 제품별 subtitle
- 제품 설명
- 주요 특징
- 주요 사양
- 다운로드 파일 연결
- 공개/비공개 여부

현재 파일:

```text
site/src/data/products.json
```

### 다운로드 자료 정리

현재 구버전 폴더의 PDF/ZIP 파일을 선별 복사했다.

확인 필요:

- 파일명이 고객에게 노출되어도 괜찮은지
- 최신 파일인지
- 중복 파일이 있는지
- 비공개 자료가 포함되어 있지 않은지
- ZIP 대신 PDF로 제공할 자료가 있는지

위치:

```text
site/public/downloads/
```

## 우선순위 2: 화면 품질 개선

### 제품 상세 이미지 개선

현재 clean 이미지는 원본 합성 이미지에서 crop/편집한 임시 개선본이다. 원본 제품 사진이 있으면 교체하는 것이 좋다.

예정 작업:

- 제품별 원본 사진 확보
- 제품 배경 통일
- 상세 이미지 비율 통일
- 카드 이미지도 필요 시 텍스트 없는 형태로 교체

### 모바일 화면 점검

확인 필요:

- 모바일 메뉴 동작
- 제품 카드 1열 표시
- 상세 페이지 이미지 크기
- 다운로드 버튼 줄바꿈
- 문의/A/S 페이지 가독성

### 디자인 세부 정리

예정 작업:

- 제품 카드 간격 보정
- 상세 페이지 사양표 가독성 개선
- 메인 hero 문구/이미지 배치 개선
- 회사소개/기술정보 페이지 시각 요소 추가

## 우선순위 3: 로컬 관리 툴 개선

현재 로컬 관리 툴은 1차 기능을 제공한다.

위치:

```text
tools/content-manager
```

실행:

```bash
cd tools/content-manager
npm start
```

주소:

```text
http://127.0.0.1:5081/
```

### 개선 예정 기능

- 제품 사양표를 JSON textarea가 아니라 행 추가/삭제 UI로 변경
- 다운로드 파일 목록을 행 추가/삭제 UI로 변경
- 이미지 미리보기 추가
- 업로드한 파일 삭제 기능 추가
- 제품 순서 drag-and-drop 또는 버튼 이동
- 제품 복제 기능
- 공지사항 미리보기
- 배포 전 변경사항 요약 표시
- 실서버 FTP 배포 기능 추가

### 주의

관리 툴은 로컬 전용이다. 운영 서버에 올리지 않는다.

## 우선순위 4: 배포 체계 정리

### 테스트 서버

현재 테스트 배포는 다음 명령으로 수행한다.

```bash
cd site
npm run validate
npm run build
npm run deploy:test
```

테스트 서버:

```text
\\192.168.0.9\web\teraleaderHomePage
```

테스트 URL:

```text
https://teraleader.synology.me:50081/
```

### 실제 서버 배포

실제 운영 서버는 inames NT i Basic 호스팅이다. Windows/IIS 계열로 예상된다.

예정 작업:

- FTP 접속 정보 확인
- 업로드 대상 루트 확인
- 기본 문서 `index.html` 확인
- `.svg`, `.webp`, `.json`, `.pdf`, `.zip` MIME 제공 여부 확인
- 실제 도메인에서 `/index.html`, `/products/`, `/sitemap.xml` 확인
- `site/scripts/deploy-prod.mjs` 또는 별도 FTP 배포 스크립트 작성

## 우선순위 5: 구버전 자료 정리

구버전 원본은 보존한다.

위치:

```text
old_src/wwwhome
```

예정 작업:

- 새 사이트에 필요한 자료와 불필요한 자료 구분
- 게시판 DB 자료를 정적 공지로 옮길지 검토
- 구버전 이미지 중 고품질 원본이 있는지 확인
- 채용 포스터 등 홈페이지에 불필요한 자료 분리

## 우선순위 6: 운영 문서 보강

추가하면 좋은 문서:

- 콘텐츠 수정 매뉴얼
- 제품 등록 매뉴얼
- 테스트 배포 매뉴얼
- 실제 운영 배포 매뉴얼
- 장애/복구 절차
- 백업 폴더 정리 기준

## 보류 또는 제외한 기능

### DB 기반 게시판

현재 제외.

이유:

- 운영 복잡도 증가
- ASP/MSSQL 의존 재발생
- 보안 관리 필요
- 현재 요구는 이메일 문의로 대체 가능

### 운영 서버 관리자 페이지

현재 제외.

이유:

- 운영 서버에 관리자 기능 노출
- 인증/권한/업로드 보안 이슈
- inames 호스팅 제약
- 로컬 전용 관리 툴로 충분히 대체 가능

### Node 서버 운영

현재 제외.

이유:

- 운영 호스팅에서 Node 실행 보장 안 됨
- 정적 사이트 배포가 더 단순함
